# M998: Upload signed firmware to USB over serial

`M998` provides a binary-safe, sequential Base64 transport for staging a BBF
on the printer's USB drive. It never flashes or reboots by itself. The final
file is `/usb/FWUPD.BBF`; invoke `M997 /usb/FWUPD.BBF` separately after a
successful upload.

The maximum BBF size is 32 MiB. Each data command carries at most 48 decoded
bytes so the complete command fits the firmware's 96-character G-code line.

```gcode
M998 P0 S<bytes> H<64-character-sha256>
M998 P1 O<decoded-offset> D<base64-data>
M998 P2
M998 P3
```

- `P0` starts a new upload, truncates `/usb/FWUPD.TMP`, and returns
  `FW_UPLOAD READY chunk=48`.
- `P1` accepts the next chunk only when `O` exactly equals the acknowledged
  decoded offset. Success returns `FW_UPLOAD OFFSET <next-offset>`.
- `P2` requires the declared byte count and SHA-256 digest to match, then
  renames the temporary file to `/usb/FWUPD.BBF` and returns
  `FW_UPLOAD COMPLETE /usb/FWUPD.BBF`.
- `P3` aborts and removes the temporary file.

Hosts must wait for each normal G-code `ok` and `FW_UPLOAD OFFSET` response
before sending the next chunk. On an offset error, resend from the last
acknowledged offset. Hash, USB-write, or internal SHA errors abort the upload
and remove the temporary file. The Prusa bootloader still validates the BBF
signature, printer model, and compatibility before flashing.
