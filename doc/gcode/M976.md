# M976: Pressure advance calibration and extrusion-health setup

For ready-to-paste PrusaSlicer and OrcaSlicer start-G-code templates, see the
[Auto PA slicer setup guide](../auto_pa_slicer_setup.md).

`M976` performs a RAM-only load-cell calibration for the currently selected
tool and logical filament. It is intended for slicer-generated start G-code.

The manual **Control > Pressure Advance Calibration** workflow enumerates only slots with an
assigned loaded material. It defaults to that material's nozzle temperature,
resets a temporary override when the selected material changes, and clamps an
override to ±15 °C around the material profile (within the firmware's global
170–300 °C calibration envelope). Sequential mode always starts from each
loaded slot's own profile.

```gcode
M976 T<physical-tool> L<logical-filament> [S<nozzle-temperature>] [B<fallback-pressure-advance>]
```

The slicer must select the requested tool or MMU filament before issuing the
command. When `S` is present, firmware sets and waits for that physical
hotend's temperature before calibrating. Without `S`, the hotend must already
be hot enough to extrude. Issue one command for every tool/filament combination
used by the job. Repeating the same `T`/`L` combination reapplies the cached
result without extruding again (and still applies/waits for `S`), which also
supports serial jobs and later MMU tool changes.

Every successful measurement or cached-result reuse also emits a replayable
`M572 D<physical-tool> S<value>` line to the serial host. The value is applied
inside firmware before that line is reported; the line is output for logging,
verification, and optional reuse rather than being queued a second time.

The existing `M572` value is used as the fallback. `B` overrides it. If neither
provides a positive value, a conservative material preset is used. A failed
calibration restores the fallback. Results and conservative material flow limits are cleared
at the next normal or serial print start and are never written to persistent
storage.

Any nozzle target supplied with `S`, including temperatures inside an `A`
batch manifest, is scoped to calibration. Firmware snapshots every hotend
target before M976 and restores all of them on success, cached-result reuse,
validation failure, measurement failure, or tool/MMU change failure. M976 does
not leave a calibration temperature active after returning.
Successful unattended batches also wait until every restorable hotend target
at or above the extrusion minimum is reached. In the supplied MMU templates,
an explicit pre-calibration `M109 R...` establishes the material-safe probing
target, so M976 finishes cooling before the following full MBL begins.

On machines with filament already loaded, homing and the local anchor
micro-mesh run at the loaded material profile's lower
`nozzle_preheat_temperature`; after probing, firmware creates 10 mm of Z
clearance so the hotend is not touching the sheet, then heats to `S`. MMU
batches probe unloaded, create the same clearance, then heat and load the requested slot. This reduces
probing ooze without weakening the extrusion measurement.

On machines with a waste bin, calibration and cleanup occur over the bin. On
other supported machines the measurement occurs in free air beyond the front
print boundary. After measurement, a four-point local micro-mesh is measured
around a dedicated front-edge slot; the strand is then anchored and wiped in
that slot. Slots are separated by logical filament number.

After physically removing an old anchor, acknowledge the clean slot with:

```gcode
M976 C L<logical-filament>
```

Firmware refuses to probe or reuse a slot still marked occupied.

## Slicer integration

Emit `M976` after selecting each used filament. `T` is the active
physical extruder and `L` is a job-local logical-filament identifier from 0 to
7. For an MMU on a single-extruder printer, keep `T0` and assign a distinct `L`
to every used filament. For an XL toolchanger, select each tool before calling
the command and normally use the same number for `T` and `L`.

```gcode
; Single material: heat tool 0 to 215 C, wait, then calibrate
M976 T0 L0 S215

; Two XL tools (select/heat commands abbreviated)
T0
M976 T0 L0 S215
T1
M976 T1 L1 S245

; Two MMU filaments on the same physical hotend
T0 ; select MMU filament 0 through the normal tool-change path
M976 T0 L0 S215
T1 ; select MMU filament 1; physical hotend remains 0
M976 T0 L1 S230
```

`M976` deliberately does not perform the `T`/MMU change itself: the preceding
standard tool-change command retains the printer's normal loading, docking,
purging and safety behavior. `T` on `M976` identifies and verifies the physical
hotend whose loadcell and heater are used; `L` identifies the loaded filament.

### One-command batch mode

Use `A` followed by a comma-separated manifest to have one command validate
and calibrate the whole job:

```gcode
M976 A 0:0:PLA:215,1:1:PETG:245
```

An eight-tool INDX job can calibrate all tools with one command. Replace each
material and temperature with the exact assignment reported by `M865 Q` and
used by the sliced job:

```gcode
M976 A 0:0:PLA:215,1:1:PLA:215,2:2:PETG:240,3:3:PETG:240,4:4:ASA:260,5:5:ASA:260,6:6:PC:275,7:7:PC:275
```

The command validates the complete eight-entry manifest first, then picks and
calibrates tools 0 through 7 in order. Omit tools not used by the job.

Each entry is `physical-tool:logical-filament:expected-material:temperature`.
Firmware validates every entry, temperature, duplicate logical slot and stored
filament assignment before heating, moving, loading or extruding anything. A
material mismatch reports `M976 invalid batch or loaded-material mismatch` and
aborts the entire command immediately. Material comparison is exact and uses
the printer's configured loaded-filament metadata; a loadcell cannot identify
polymer chemistry independently.

On an XL/toolchanger, batch mode docks and picks each physical tool through the
normal tool-change implementation. On an MMU printer, physical tool is always
`0`; logical filament selects the MMU slot. The calibration MMU change explicitly
unloads before every home/local probe, performs the load-to-nozzle recovery
workflow afterward, and limits
the final load prime to 2 mm so it cannot form the normal long purge loop over
the bed:

```gcode
M976 A 0:0:PLA:215,0:2:PETG:235,0:4:PA:285
```

The command returns only after every requested entry is calibrated or after
the first tool-change/calibration failure. Toolchanger batches leave the last
requested physical tool selected. MMU batches unload the final filament so the
slicer's following full MBL runs with an empty nozzle; the normal post-MBL `T`
command reloads the initial print filament.

After validation, serial hosts receive `PA_CALIBRATION batch accepted entries=N`
and regular `echo:busy: processing` keepalives throughout the blocking phases.
If a numbered host nevertheless retransmits the exact M976 line that is still
executing, firmware discards that duplicate and emits another busy response
instead of flushing RX and entering a forward `Resend:` loop. The one final
`ok` still belongs to the original command.

## Manual calibration from the printer

Open **Control > Pressure Advance Calibration**, immediately above **Calibrations & Tests**. The dock-calibration-style screen lets
you toggle any assigned physical tool or MMU slot, choose the material's
default nozzle temperature or a temporary override, and press one
**Run Selected Calibrations** action. Each loaded tool has its own Material/default
or explicit temperature control. The UI sends one
`M976 M K<mask> U<t0>,<t1>,<t2>,<t3>,<t4>,<t5>,<t6>,<t7>` batch; zero uses that tool's
material temperature and explicit values are clamped to its ±15 °C safety range.
`S` optionally carries the shared, per-material-clamped temperature override.
The progress screen takes over when the command starts, blocks other printer UI,
keeps chamber lighting active, remains visible over file and serial prints, and
exposes only Abort until the aggregated manual results are ready. Abort
restores the G-code/profile fallback. All MMU runs unload before probing and
unload the final calibrated slot before returning or presenting results. A completed manual run shows every selected PA value
and lets the user save all corresponding `M572` lines to
`/usb/pa-calibration.gcode` or exit. After Save or exit, firmware raises to the
normal safe clearance and parks the toolhead at the machine's standard park
position.

Manual runs display an additional confirmation requiring the build plate,
front service area, and existing PA anchor locations to be clear. Confirming
the prompt acknowledges the selected anchor slots before running. This prompt
and automatic anchor acknowledgement are UI-only; unattended slicer G-code
continues to reject occupied slots until they are explicitly cleared.

The PA excitation follows the PrusaPATuner method: a continuous asymmetric
0.8/8.0 mm/s filament square wave is measured at the loadcell sample rate.
Executed E-step positions locate the real transitions. Firmware first searches
around the fallback and then refines the best response in 0.002-second PA
increments. The score combines normalized transition error area,
over/undershoot and settling time.

`M976` does not claim to calibrate maximum flow from a short ramp. It applies a
conservative material ceiling to the planner and uses the known-good PA burst
to arm a RAM-only pressure model. During the print, sustained forward E motion
with no pressure rise, a drastic pressure collapse, or a sustained pressure
breakout above the healthy flow curve injects `M1601` and enters the normal
stuck-filament pause/recovery workflow. Monitoring latches off after a fault to
avoid repeated pauses and is armed again by the next `M976` or print job.

The monitor uses executed stepper position, not merely queued G-code. It does
not turn the extruder into a physical filament encoder: severe slip is inferred
from nozzle pressure, while the normal filament sensors remain responsible for
presence/runout detection. During the deliberately pulsed calibration
excitation, sensor sampling stays active but runout/autoload event delivery is
temporarily locked; normal event handling is restored before M976 returns or a
manual MMU unload begins.

The excitation and step-response decomposition are firmware-side adaptations
of the experimental [CNCKitchen PrusaPATuner](https://github.com/CNCKitchen/PrusaPATuner)
research approach. Exact firmware timestamps eliminate its UDP alignment and
packet-loss uncertainty.
